┌───────────────────────────────────────────────────────────────────┐
│  METHOD 1: PowerShell Script (Semi-Automatic) ⭐ RECOMMENDED     │
└───────────────────────────────────────────────────────────────────┘

  In PowerShell, run:
  
    .\auto-fix.ps1

  The script will:
  ✓ Pause and ask you to fix database in MySQL Workbench
  ✓ Clean old compiled files automatically
  ✓ Start the application automatically


┌───────────────────────────────────────────────────────────────────┐
│  METHOD 2: Manual Fix (Full Control) ⚙️                          │
└───────────────────────────────────────────────────────────────────┘

  STEP 1 - Fix Database:
    1. Open MySQL Workbench
    2. Open file: FIX_DATABASE.sql  
    3. Click Execute (⚡)

  STEP 2 - Clean and Run:
    In PowerShell:
      .\mvnw.cmd clean
      .\mvnw.cmd spring-boot:run

    OR double-click:
      START_APP.bat


┌───────────────────────────────────────────────────────────────────┐
│  METHOD 3: Super Quick (If you know what you're doing) 🚀        │
└───────────────────────────────────────────────────────────────────┘

  1. MySQL Workbench → Run this SQL:
     
     USE itsm_db;
     ALTER TABLE users MODIFY COLUMN role VARCHAR(20) NOT NULL;

  2. PowerShell → Run:
     
     .\START_APP.bat

║                    ❓ WHAT ARE THE ERRORS?                        ║
╚═══════════════════════════════════════════════════════════════════╝

  ERROR #1: Database column 'role' is too small
            → Can't fit "ENGINEER" (8 chars) or "FACULTY" (7 chars)
            → Fix: Expand to VARCHAR(20)

  ERROR #2: Old compiled files have wrong package declarations
            → Spring tries to load old .class files → crash
            → Fix: Run "mvnw clean" to delete them
