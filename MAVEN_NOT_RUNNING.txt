📌 THE PROBLEM:
───────────────

Maven keeps failing with 2 errors:

  1. Database error: "Data truncated for column 'role'"
  2. Compilation error: "Package mismatch"


🎯 THE ROOT CAUSE:
──────────────────

  1. Your database 'users' table has a 'role' column that's TOO SMALL
     → It can't fit "ENGINEER" or "FACULTY" text

  2. Your 'target' folder has OLD compiled .class files
     → These were compiled before we fixed the package names
     → Maven uses these old files instead of recompiling

❓ WHY DOESN'T "mvnw clean" WORK?
─────────────────────────────────

  Sometimes Maven's clean doesn't delete everything properly.
  The OLD .class files stay in target/classes/
  These files have WRONG package declarations
  Spring Boot tries to load them → CRASH!

  SOLUTION: Manually delete the entire 'target' folder
