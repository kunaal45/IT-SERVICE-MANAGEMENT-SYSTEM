═══════════════════════════════════════════════════════════════════════════


🔄 THE WORKFLOW:
═══════════════════════════════════════════════════════════════════════════

1. FACULTY creates Issue
   ↓ (automatic)
2. System creates Ticket (status: OPEN)
   ↓ (ADMIN assigns)
3. Ticket → ASSIGNED
   ↓ (ENGINEER starts)
4. Ticket → IN_PROGRESS
   ↓ (ENGINEER resolves)
5. Ticket → RESOLVED
   ↓ (FACULTY closes)
6. Ticket → CLOSED


═══════════════════════════════════════════════════════════════════════════


📋 API ENDPOINTS:
═══════════════════════════════════════════════════════════════════════════

Create Issue:     POST   /api/issues
My Issues:        GET    /api/issues/my
All Issues:       GET    /api/issues (Admin)

Assign Ticket:    PUT    /api/tickets/{id}/assign (Admin)
Start Work:       PUT    /api/tickets/{id}/start (Engineer)
Add Notes:        PUT    /api/tickets/{id}/notes (Engineer)
Resolve:          PUT    /api/tickets/{id}/resolve (Engineer)
Close:            PUT    /api/tickets/{id}/close (Faculty)

Get Tickets:      GET    /api/tickets (role-based)
Dashboard:        GET    /api/tickets/dashboard
