🎯 KEY OBSERVATIONS:
═══════════════════════════════════════════════════════════════════════════

Role Differences:
  
  👔 ADMIN sees:
     → ALL 20 tickets
     → Can assign to anyone
     → Full system overview
     → All users visible
  
  👷 ENGINEER sees:
     → Only tickets assigned TO THEM
     → Can update status/notes
     → Their specific workload
     → Cannot see others' tickets
  
  🎓 FACULTY sees:
     → Only tickets THEY created
     → Cannot assign to engineers
     → Can track their issues
     → Cannot see others' tickets


Workflow:
  1. Faculty creates ticket → Status: OPEN
  2. Admin assigns to engineer → Status: ASSIGNED
  3. Engineer starts work → Status: IN_PROGRESS
  4. Engineer fixes issue → Status: RESOLVED
  5. Admin/Faculty closes → Status: CLOSED
