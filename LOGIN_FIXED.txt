🎯 THE ISSUE:
═══════════════════════════════════════════════════════════════════════════

When you tried to login and access engineer or admin portal, you got:
  "localhost access was denied"

REASON: The JavaScript login handler was incorrectly parsing the API response.

The API returns:
  {
    "success": true,
    "data": { token, email, name, role, userId }
  }

But JS was accessing: data.token (WRONG)
Should access: data.data.token (CORRECT)


═══════════════════════════════════════════════════════════════════════════


✅ THE FIX:
═══════════════════════════════════════════════════════════════════════════

File: /src/main/resources/static/js/app.js
Function: handleLogin()

Changed:
  ❌ const data = await response.json();
     localStorage.setItem(TOKEN_KEY, data.token);

✓ const apiResponse = await response.json();
  const data = apiResponse.data;
  localStorage.setItem(TOKEN_KEY, data.token);

Result: Correct parsing of nested API response
